# Paperweight

**Make your phone a paperweight.** Paperweight turns an iPhone into a calm, quiet object on a schedule you set — distracting apps are shielded outside the windows you choose, and the only way back early is a deliberate tap of a physical NFC token you've placed somewhere inconvenient.

An iOS focus tool built on Apple's Screen Time (Family Controls) framework. Everything runs on-device; no account, no tracking.

· [Support](https://paperweight.baltar.media/) · [Privacy](https://paperweight.baltar.media/privacy.html)

## How it works

- **Schedule** — paint a weekly grid of free time. Outside those windows, your selected apps and categories are shielded. Restrictions apply and lift automatically.
- **NFC unlock** — tap your registered NFC token for a short, timed unlock, or to turn Paperweight off. Tapping the physical tag *is* the confirmation — that small bit of friction is the point.
- **Never locked out** — single-use recovery codes, an optional timed "cool-off" unlock that releases after a day, and the ultimate failsafe: deleting the app clears every restriction (we never set `denyAppRemoval`).

## Architecture

- **SwiftUI**, iOS 17+, dark-only ("Quiet Glass" design language).
- **FamilyControls / ManagedSettings / DeviceActivity** — app selection, shields, and scheduled enforcement via a `DeviceActivityMonitor` app extension.
- **CoreNFC** — reads the token's hardware UID (`NFCTagReaderSession`); tags need no programming.
- **CryptoKit** — SHA-256 hashing for recovery codes (stored hashed, never in plaintext).
- **App Group** (`group.media.baltar.paperweight`) — shares config between the app and the monitor extension.

```
Paperweight/            iOS app (views, services, app entry)
PaperweightMonitor/     DeviceActivity monitor extension
Shared/                 models, view-models, services, theme (compiled into both)
PaperweightTests/       unit tests (standalone, no host app)
docs/                   GitHub Pages: support + privacy
project.yml             XcodeGen project definition (source of truth)
```

## Building

The Xcode project is **generated by [XcodeGen](https://github.com/yonaskolb/XcodeGen)** from `project.yml` and is not committed.

```sh
brew install xcodegen
xcodegen generate
open Paperweight.xcodeproj
```

Then set your own **Development Team** in *Signing & Capabilities* (it's intentionally blank in `project.yml`). The Family Controls entitlement and NFC require a **physical device** — the app can't run in the Simulator.

```sh
# run the unit tests (these do run in the Simulator)
xcodebuild -project Paperweight.xcodeproj -scheme PaperweightTests \
  -destination 'platform=iOS Simulator,name=iPhone 16 Pro' test
```
